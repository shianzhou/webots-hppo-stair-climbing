
日志中应当记录的内容

日志应当按照总任务周期去记录，也就是有一个总的episod_num，当前这种状态，可以暂时保持同时含有总周期数和单独的周期数

总周期是指完整的动作周期，从choose_action得到值之后就不会在进行得到值了，之后无非就是抓取或者踩踏，所以决策然后行动是一个完整的周期

底层的周期数就是，连续记录抓取多少次，第一次决策抓取，那抓取就进行了一次，第二次没决策抓取，那就不进行，第三次还是抓取，抓取周期加一

- loss值，包括离散的loss，连续的loss
- 每个动作周期的奖励值，决策奖励，抓取奖励，踩踏奖励和总奖励
- 动作类型，一个完整动作周期（即智能体给出动作后，根据决策去执行动作，动作执行完），当前的动作类型，比如决策给出的是0，我们认为是抓取，给出的是1，我们认为是抬腿，记录当前
- 记录时间
- 总周期数
