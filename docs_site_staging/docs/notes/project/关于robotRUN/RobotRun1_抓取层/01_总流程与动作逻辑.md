# RobotRun1 抓取层：总流程与动作逻辑

对应代码：`E:\project_MultiAgent_h_change\python_scripts\PPO\RobotRun1.py`

## 一句话定位

`RobotRun1` 负责抓取阶段的动作执行。它把抓取模型输出的 `discrete_action + continuous_action` 映射成左右肩和左右臂的电机目标，执行到位后通过抓爪触碰、闭合抓爪、GPS 台阶匹配和安全约束判断抓取是否成功。

## 输入参数从哪里来

主训练流程中，抓取阶段调用：

```python
action_dict = hppo_agent.choose_action(obs=obs, x_graph=robot_state)
d_action = action_dict['discrete_action']
c_action = action_dict['continuous_action']
next_state, reward, done, catch_success = env.step(d_action, c_action, steps)
```

`env.step(...)` 内部会创建/调用 `RobotRun1.RobotRun`。

输入动作包括：

| 参数 | 含义 |
| --- | --- |
| `discrete_action` | 2 维离散门控，控制肩和臂是否采用新动作 |
| `continuous_action` | 2 维连续动作，分别对应肩部和手臂 |
| `step` | 当前抓取阶段步数 |

## 动作如何映射

### 1. 离散门控

`_map_policy_actions()` 中：

```python
mapped_shoulder = prev_shoulder if int(d0) == 0 else cur_shoulder
mapped_arm = prev_arm if int(d1) == 0 else cur_arm
```

含义：

| 离散值 | 含义 |
| --- | --- |
| `0` | 沿用上一时刻动作 |
| `1` | 使用本步新的连续动作 |

### 2. 连续动作到目标角度

抓取层控制左侧肩/臂，再镜像到右侧：

```python
left_arm_target = 1.25 * action_arm + 0.25
left_shoulder_target = 0.2995 * action_shoulder - 0.145
```

随后转成相对当前关节角度的增量：

```python
ArmLower = left_arm_target - current_left_arm
Shoulder = left_shoulder_target - current_left_shoulder
```

并限制单步变化：

```python
ArmLower = clamp(ArmLower, -0.3, 0.3)
Shoulder = clamp(Shoulder, -0.3, 0.3)
```

### 3. 目标关节

最终四个上肢目标是：

```python
next = [
    robot_state[1] + Shoulder,  # 左肩 ShoulderL
    robot_state[0] - Shoulder,  # 右肩 ShoulderR
    robot_state[5] + ArmLower,  # 左臂 ArmLowerL
    robot_state[4] - ArmLower,  # 右臂 ArmLowerR
]
```

所以抓取动作是左右对称执行的。

## 总执行流程

```text
模型输出 discrete_action + continuous_action
  -> _map_policy_actions() 处理门控
  -> 计算 left_arm_target / left_shoulder_target
  -> 转成 Shoulder / ArmLower 增量
  -> 写入 future_state 和 next
  -> _apply_upper_body_action() 下发电机目标
  -> _wait_action_settle() 等待关节到位
  -> 检查关节限位、IMU、碰撞
  -> 抓爪触碰后 _close_grasp_and_read_pair()
  -> 重新计算 GPS 台阶匹配
  -> compute_reward()
  -> 返回 next_state, reward, done, catch_success
```

## 成功/失败/终止判定

### 成功

抓取成功需要两个条件：

1. 抓爪闭合后触碰模式符合 `Darwin_config.touch_T`。
2. GPS 台阶匹配 `is_target_stair == 1`。

此时：

```python
catch_success = True
catch_state = 'grasped_right'
```

### 抓错

如果抓爪闭合成功，但 GPS 台阶匹配失败：

```python
catch_state = 'grasped_wrong'
catch_success = False
```

### 抓取失败

如果闭合后触碰模式符合失败模板，或无法形成完整抓取：

```python
catch_state = 'attempt_failed'
```

### 约束终止

以下情况会提前终止：

- 关节越界。
- IMU 超出安全范围。
- 手臂/腿部碰撞传感器触发数达到阈值。
- 动作执行后关节跟踪误差过大。

## 奖励逻辑

`compute_reward()` 主要包含：

| 奖励项 | 作用 |
| --- | --- |
| 距离变化奖励 | 鼓励靠近目标台阶 |
| 接近奖励 | 距离小于一定范围时加分 |
| 动作惩罚 | 动作太小或太大都惩罚 |
| 时间惩罚 | 步数越多惩罚越大 |
| 碰撞惩罚 | 避免手臂/腿碰撞 |
| 终局奖励 | 抓对目标大奖励，抓错小奖励或失败惩罚 |

## 容易混淆的点

1. `continuous_action` 不是直接电机角度，要先映射为目标角度，再变成增量。
2. 抓取成功不只看抓爪触碰，还要看 GPS 是否对应目标台阶。
3. `catch_flag` 不是固定步数决定，而是根据抓爪触碰传感器动态决定。
4. 左侧动作会镜像到右侧关节。
