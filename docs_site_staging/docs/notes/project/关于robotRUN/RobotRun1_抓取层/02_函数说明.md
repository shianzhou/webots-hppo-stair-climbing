# RobotRun1 抓取层：函数说明

对应代码：`RobotRun1.py`

## `__init__(robot, discrete_action, continuous_action, step)`

初始化本步抓取动作执行器。

主要工作：

1. 读取当前 `robot_state` 和 GPS。
2. 保存模型输出的离散/连续动作。
3. 调用 `_map_policy_actions()` 处理离散门控。
4. 把连续动作映射成左右肩/臂目标。
5. 初始化抓取触碰传感器、碰撞传感器、`future_state`、`next_state` 和返回标志。

## `_to_float(value, default=0.0)`

安全地把模型输出转成 Python float。

用于兼容：

- 普通数值。
- numpy 标量。
- torch 标量。
- `None` 或异常值。

## `_map_policy_actions(discrete_action, continuous_action)`

处理抓取模型的 2 维离散门控和 2 维连续动作。

规则：

```text
离散为 0：沿用上一动作
离散为 1：使用当前连续动作
```

它还会更新类变量：

```python
RobotRun._prev_shoulder_action
RobotRun._prev_arm_action
```

## `_compute_distance_metrics()`

根据 GPS 计算目标台阶匹配分数。

输出：

```python
stair_score_1, stair_score_2, distance, is_target_stair
```

其中 `is_target_stair` 由两个台阶评分之和决定，用来判断是否抓到目标台阶。

## `_check_joint_limits()`

检查 `future_state` 中每个关节是否在 `Darwin_config.limit` 范围内。

如果任一关节越界，返回 `False`。

## `_check_imu_limits()`

读取加速度计和陀螺仪，判断机器人姿态是否在安全范围内。

如果任一轴超过配置阈值，返回 `False`。

## `_apply_upper_body_action()`

真正把四个上肢目标下发到 Webots 电机：

```python
motors[1].setPosition(next[0])  # ShoulderL
motors[0].setPosition(next[1])  # ShoulderR
motors[5].setPosition(next[2])  # ArmLowerL
motors[4].setPosition(next[3])  # ArmLowerR
```

这是抓取阶段的核心动作执行函数。

## `_wait_action_settle(max_steps=100, tol=0.01)`

等待上肢四个关键关节到达目标位置。

作用是避免刚下发动作、关节还没动到位时，就立即读取触碰或 GPS 进行判定。

## `_has_any_grasp_touch()`

检查 6 个抓取触碰传感器中是否有任意一个触发：

```text
grasp_L1, grasp_L1_1, grasp_L1_2, grasp_R1, grasp_R1_1, grasp_R1_2
```

如果触发，说明抓爪接触到物体，可以进入闭合验证。

## `_close_grasp_and_read_pair(wait_ms=2000)`

闭合左右抓爪，并读取左右抓取触碰结果。

主要步骤：

1. 打印闭合前抓取传感器值。
2. 设置 `GraspL/GraspR` 电机位置为 `-0.5`。
3. 等待一段时间让抓爪闭合。
4. 汇总左侧所有 `grasp_L*` 传感器最大值。
5. 汇总右侧所有 `grasp_R*` 传感器最大值。
6. 和 `Darwin_config.touch_T / touch_F` 比较，得到 `success` 和 `failed`。

返回：

```python
success, failed
```

## `_check_collision_constraint()`

检查手臂和腿部碰撞传感器。

当前逻辑是：至少 2 个碰撞传感器触发才算真正碰撞。

这样比“任意一个触发就失败”更不敏感。

## `_refresh_next_state()`

读取当前机器人关节状态，写入 `next_state`。

用于返回给 PPO 作为动作后的状态。

## `_check_tracking_constraint(tol=0.005)`

比较执行后的关节传感器值和 `future_state`。

如果偏差过大，说明动作没有按目标执行，返回 `False`。

## `_log_constraint_trigger(name, details='')`

统一打印约束触发日志。

便于训练时快速定位是关节、IMU、碰撞还是抓取结果导致终止。

## `compute_reward(...)`

统一计算抓取阶段 reward。

输入包括：

- `distance`
- `prev_distance`
- `done`
- `success`
- `failed`
- `goal`
- `collision`
- `imu_ok`
- `joints_ok`

逻辑包括距离变化奖励、接近奖励、动作惩罚、时间惩罚、碰撞惩罚和终局奖励。

## `run()`

抓取阶段单步执行入口。

总流程：

1. 推进 Webots 一步。
2. 初始化返回值。
3. 计算当前 GPS 距离和台阶评分。
4. 如果还没触碰，执行上肢动作并等待到位。
5. 检查关节、IMU、碰撞、跟踪误差。
6. 如果触碰抓爪，闭合抓爪并判断成功/失败。
7. 重新计算距离和 reward。
8. 刷新 `next_state`。
9. 返回：

```python
next_state, reward, done, catch_success
```
