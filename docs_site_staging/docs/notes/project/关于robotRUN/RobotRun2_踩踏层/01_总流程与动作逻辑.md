# RobotRun2 踩踏层：总流程与动作逻辑

对应代码：`E:\project_MultiAgent_h_change\python_scripts\PPO\RobotRun2.py`

## 一句话定位

`RobotRun2` 负责踩踏阶段的动作执行。它把踩踏模型输出的三维连续动作映射为左腿三个关节 `LegUpperL / LegLowerL / AnkleL` 的目标角度，执行到位后读取左脚 GPS 和触碰传感器，判断是否踩到目标台阶。

## 输入参数从哪里来

主流程中，踩踏阶段调用：

```python
tai_dict = tai_agent.choose_action(obs=tai_state, x_graph=tai_state)
```

动作经过门控后传入：

```python
env.step2(
    robot_state,
    action_leg_upper,
    action_leg_lower,
    action_leg_ankle,
    steps,
    catch_flag,
    gps_values[4],
    gps_values[0], gps_values[1], gps_values[2], gps_values[3],
)
```

`env.step2(...)` 内部会调用 `RobotRun2.RobotRun`。

## 动作如何映射

踩踏模型输出三个连续动作：

| 动作 | 控制关节 |
| --- | --- |
| `action_leg_upper` | `LegUpperL`，索引 11 |
| `action_leg_lower` | `LegLowerL`，索引 13 |
| `action_ankle` | `AnkleL`，索引 15 |

在 `__init__` 中先映射为关节增量：

```python
leg_upper_delta = 1.09 * action_leg_upper + 0.59
leg_lower_delta = 1.14 * action_leg_lower - 1.11
ankle_delta = 1.305 * action_ankle - 0.085
```

然后 `_compute_targets()` 计算电机目标角度：

```python
targets = [
    robot_state[11] + leg_upper_delta,
    robot_state[13] + leg_lower_delta,
    robot_state[15] + ankle_delta,
]
```

所以：

```text
target = 当前关节角度 + 动作映射后的 delta
```

这里的 `target` 是关节角度目标，不是台阶坐标。台阶坐标是 `gps_goal1`。

## 总执行流程

```text
模型输出三维动作
  -> 主流程门控得到最终执行动作
  -> RobotRun2.__init__ 映射为 delta
  -> _compute_targets() 得到三个 target
  -> _apply_joint_limits() 限制 target
  -> _check_joint_limits() / _check_imu_limits()
  -> _set_leg_targets() 下发 Webots 电机目标
  -> _wait_leg_target_reached() 等待真实到位
  -> _read_step_distance_after_action() 读取动作后左脚 GPS 距离
  -> _read_left_foot_touch() 检查左脚触碰
  -> 根据触碰 + 距离判断 goal
  -> _compute_reward()
  -> 返回 next_state, reward, done, good, goal, count
```

## 成功/失败/终止判定

### 成功

成功条件：

```text
左脚触碰传感器触发 + foot_gps1 到 gps_goal1 的距离 <= success_dist_thresh
```

当前阈值：

```python
success_dist_thresh = 0.04
```

成功后：

```python
goal = 1
done = 1
```

### 偏踩

如果左脚触碰了，但距离大于阈值：

```python
goal = 0
done = 1
```

日志会打印 `STEP_OFF_TARGET`。

### 无触碰

如果没有左脚触碰：

```python
done = 0
```

继续下一步，直到主流程达到最大步数。

### 约束终止

以下情况会提前终止：

- 关节限位检查失败。
- IMU 超出阈值。
- 手臂/腿部碰撞传感器触发。

## 奖励逻辑

`_compute_reward()` 包含：

| 奖励项 | 作用 |
| --- | --- |
| 距离惩罚 | 距离目标越远惩罚越大 |
| 距离进度奖励 | 比上一步更接近目标则加分 |
| 中心误差惩罚 | 对横向偏差加权，减少偏踩 |
| 远距离惩罚 | 距离超过 0.25 时额外惩罚 |
| 动作幅度惩罚 | 防止动作过大抖动 |
| 步数惩罚 | 鼓励更快踩到目标 |
| 成功奖励 | `goal=1` 时给大额奖励 |
| 偏踩惩罚 | 触碰但距离不达标时惩罚 |
| 无触碰惩罚 | 回合结束但未触碰时惩罚 |

## 容易混淆的点

1. `target` 是关节目标角度，不是台阶目标。
2. 台阶目标是 `gps_goal1`。
3. `_wait_leg_target_reached()` 只是等电机执行到位，不判断踩踏成功。
4. “落脚”不是单独函数，而是多步关节目标执行后的物理结果。
5. 成功必须同时看左脚触碰和 GPS 距离。
