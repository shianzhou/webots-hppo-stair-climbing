# RobotRun2 踩踏层：函数说明

对应代码：`RobotRun2.py`

## `__init__(...)`

初始化一次踩踏动作执行。

主要工作：

1. 保存当前 `robot_state`、动作和 GPS。
2. 把三维动作映射为三个关节增量：
   - `leg_upper_delta`
   - `leg_lower_delta`
   - `ankle_delta`
3. 设置奖励系数、成功阈值、等待到位参数。
4. 初始化 `future_state`、`next_state`。
5. 调用 `_compute_targets()` 得到三个左腿目标角度。
6. 调用 `_init_devices()` 初始化电机、GPS、触碰传感器。

## `_init_devices()`

初始化 Webots 设备：

- 22 个电机和位置传感器。
- 加速度计 `Accelerometer`。
- 陀螺仪 `Gyro`。
- 左脚 GPS `foot_gps1`。
- 左脚触碰传感器：`touch_foot_L1 / L2 / L3`。
- 碰撞传感器：手臂和腿部触碰。

## `_compute_targets()`

根据当前关节状态和动作增量计算目标角度。

核心公式：

```python
targets = [
    robot_state[11] + leg_upper_delta,
    robot_state[13] + leg_lower_delta,
    robot_state[15] + ankle_delta,
]
```

并写入：

```python
future_state[11]
future_state[13]
future_state[15]
```

## `_clamp_joint(idx, value)`

把某个关节角度裁剪到允许范围内。

输入：

- `idx`：关节索引。
- `value`：目标角度。

输出：裁剪后的角度。

## `_apply_joint_limits()`

对三个左腿目标关节执行限位：

```text
LegUpperL / LegLowerL / AnkleL
```

如果某个目标角度超出限制，会打印 `[CLAMP]` 日志。

返回裁剪后的三个目标角度。

## `_check_joint_limits()`

检查 `future_state` 中三个左腿目标角度是否都在限制范围内。

由于 `_apply_joint_limits()` 已经裁剪，这里更像安全兜底。

## `_check_imu_limits()`

读取 IMU：

- 加速度计。
- 陀螺仪。

判断是否在预设阈值内。如果超出，返回 `False` 并终止本步。

## `_set_leg_targets(target)`

真正下发三个左腿电机目标：

```python
motors[11].setPosition(target[0])
motors[13].setPosition(target[1])
motors[15].setPosition(target[2])
```

对应：

| target | 关节 |
| --- | --- |
| `target[0]` | `LegUpperL` |
| `target[1]` | `LegLowerL` |
| `target[2]` | `AnkleL` |

## `_wait_leg_target_reached(target, timeout=None, tolerance=None)`

等待三个左腿关节实际到达目标角度。

流程：

1. 推进 Webots 仿真。
2. 读取 `LegUpperL / LegLowerL / AnkleL` 位置传感器。
3. 比较当前位置和 `target`。
4. 三个误差都小于 `tolerance`，返回 `True`。
5. 超过 `timeout` 仍不到位，返回 `False`。

它不判断是否踩到台阶，只保证动作实际执行过。

## `_read_step_distance_after_action()`

动作执行后读取左脚 GPS：

```python
foot_gps_now = foot_gps1.getValues()
```

计算左脚到目标台阶 `gps_goal1` 的二维距离：

```python
distance = sqrt((goal_x - foot_x)^2 + (goal_y - foot_y)^2)
```

## `_has_collision()`

检查手臂和腿部碰撞传感器是否触发。

如果任意碰撞传感器值大于 0，返回 `True`。

## `_has_foot_contact()`

检查左脚三个触碰传感器是否任意触发。

用于判断脚底是否接触台阶或其他物体。

## `_read_left_foot_touch(wait_ms=2000)`

读取左脚触碰结果。

流程：

1. 打印触碰前传感器值。
2. 如果没有脚底接触，直接返回 `False`。
3. 等待一段时间，让接触状态稳定。
4. 读取 `touch_foot_L1 / L2 / L3`。
5. 只要任意值大于 0，返回 `True`。

## `_compute_reward(distance, foot_x, foot_y, done, goal, collision, touch_success)`

计算踩踏 reward。

主要输入：

- `distance`：左脚到目标台阶距离。
- `foot_x / foot_y`：左脚动作后位置。
- `done`：本步是否结束。
- `goal`：是否踩踏成功。
- `collision`：是否碰撞。
- `touch_success`：左脚是否触碰。

主要逻辑：

- 距离越远惩罚越大。
- 比上一步更近则奖励。
- 横向偏差越大惩罚越大。
- 动作幅度和步数都有轻微惩罚。
- `goal=1` 给成功奖励。
- 触碰但偏离目标给偏踩惩罚。
- 结束但无触碰给无触碰惩罚。

## `_refresh_next_state()`

读取 20 个电机传感器值，写入 `next_state`。

用于返回给 PPO 作为环境下一状态。

## `_finish(reward, done, good, goal, count)`

统一收尾函数。

先刷新 `next_state`，再返回：

```python
next_state, reward, done, good, goal, count
```

## `run()`

踩踏阶段单步执行入口。

完整流程：

1. 推进仿真一步。
2. 第 0 步时清空历史距离和中心误差。
3. 对目标关节限位。
4. 检查关节和 IMU。
5. 下发电机目标。
6. 等待关节到位。
7. 读取动作后左脚 GPS。
8. 检查碰撞。
9. 检查左脚触碰。
10. 如果触碰，判断距离是否小于 `success_dist_thresh`。
11. 计算 reward。
12. 更新历史距离和中心误差。
13. 返回 `next_state, reward, done, good, goal, count`。
