# RobotRun0 决策层：函数说明

对应代码：`RobotRun0.py`

## `__init__(...)`

保存决策层训练需要的对象和路径：

| 参数 | 含义 |
| --- | --- |
| `decision_agent` | 上层决策 PPO 模型 |
| `training_manager` | 控制何时 learn 的训练管理器 |
| `log_writer_decision` | 决策日志写入器 |
| `log_file_latest_decision` | 当前决策日志文件 |
| `decision_checkpoint_dir` | 决策模型保存目录 |
| `save_interval` | 保存 checkpoint 的间隔 |

它只做对象保存，不执行决策，也不执行环境动作。

## `choose_action(decision_state_input, pressure_detected=None)`

作用：把 6 维决策状态输入模型，得到上层离散决策。

核心代码：

```python
decision_dict = self.decision_agent.choose_action(obs=decision_state_input)
decision = int(decision_dict["discrete_action"][0])
```

返回：

```python
decision_dict, decision, decision_state
```

其中：

- `decision_dict`：模型输出，包含 `discrete_action`、`discrete_log_prob`、`value`。
- `decision`：转成整数后的决策，`0=catch`，`1=tai`。
- `decision_state`：三分支兼容格式 `(state, state, state)`，用于经验缓存。

## `judge_route(decision, catch_success)`

作用：把模型输出的 `decision` 转换成真正的阶段路线。

路线有三种：

| route | 含义 |
| --- | --- |
| `catch` | 进入抓取阶段 |
| `tai` | 进入踩踏阶段 |
| `re_decide` | 不进入子阶段，重新决策 |

关键约束：踩踏必须依赖 `catch_success=True`。

## `_compute_reward(decision, route, pre_branch_catch_success, post_branch_catch_success)`

作用：给上层决策模型计算奖励。

它不关心抓取/踩踏内部动作细节，只关心：

```text
当前是否已经抓取成功
模型选择了抓取还是踩踏
分支执行后抓取状态有没有变好
```

典型奖励：

- 已抓取还选抓取：错误，`-15.0`。
- 未抓取选踩踏：错误，`-10.0`。
- 已抓取选踩踏：正确，`+10.0`。
- 未抓取选抓取且成功：正确，`+5.0`。

## `finalize(...)`

作用：在一个上层决策周期结束后，完成决策层训练闭环。

主要步骤：

1. 调用 `_compute_reward()` 得到 `decision_reward`。
2. 调用 `decision_agent.store_transition()` 保存决策经验。
3. 通过 `training_manager` 判断是否调用 `decision_agent.learn()`。
4. 写入决策日志。
5. 按 `save_interval` 保存 `decision_hppo_*.ckpt`。

测试/推理模式下通常不需要调用 `finalize()`，或者需要给它加 `train_decision=False`，跳过 `store_transition()`、`learn()` 和 checkpoint 保存。
