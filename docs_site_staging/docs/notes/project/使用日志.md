# 网页使用手册

这份手册说明怎么把 Obsidian 里的笔记发布到项目网页里。你以后主要只需要改一个文件：

```text
E:\project_MultiAgent_h_change\docs_site_staging\publish.yml
```

Obsidian 原始笔记不会被网页脚本修改。脚本只会把你选择发布的内容复制到：

```text
E:\project_MultiAgent_h_change\docs_site_staging\docs\notes
```

## 1. 最重要的概念

网站有两层目录：

1. Obsidian 原始目录：你平时写笔记的地方。
2. 网站发布目录：给 MkDocs 生成网页用的副本。

你通过 `publish.yml` 决定哪些笔记进入网站。

## 2. publish.yml 每一项是什么意思

一条发布规则长这样：

```yaml
- source: "项目重构/总体文件概览.md"
  dest: "project/总体文件概览.md"
  title: "总体文件概览"
  nav: "项目代码说明"
  enabled: true
```

字段含义：

| 字段 | 意思 |
| --- | --- |
| `source` | Obsidian 里的原始笔记或文件夹，相对路径从“强化学习”目录开始写 |
| `dest` | 发布到网站里的位置，相对路径从 `docs/notes` 开始写 |
| `title` | 左侧目录里显示的名字 |
| `nav` | 左侧目录分组，相同 `nav` 会放在同一组下面 |
| `enabled` | `true` 表示发布，`false` 表示不发布 |

## 3. 发布一篇笔记

比如你想发布这篇 Obsidian 笔记：

```text
D:\obsidian\计划与随笔\2026年\强化学习\项目重构\关于网络\状态输入.md
```

在 `publish.yml` 的 `entries:` 下面写：

```yaml
- source: "项目重构/关于网络/状态输入.md"
  dest: "project/关于网络/状态输入.md"
  title: "状态输入"
  nav: "项目代码说明"
  enabled: true
```

然后运行同步命令：

```powershell
cd E:\project_MultiAgent_h_change
python docs_site_staging\scripts\sync_obsidian.py
```

## 4. 不发布或撤回一篇笔记

把这一项改成：

```yaml
enabled: false
```

然后重新运行：

```powershell
python docs_site_staging\scripts\sync_obsidian.py
```

脚本会重建 `docs/notes`，所以被关闭的内容会从网站副本里消失。

## 5. 让几篇内容在左侧同一个目录下面

只要它们的 `nav` 一样，就会在左侧同一组下面。

例如：

```yaml
- source: "项目重构/训练流程/PPO_episoid_1训练流程说明书.md"
  dest: "training/PPO_episoid_1训练流程说明书.md"
  title: "PPO_episoid_1 训练流程"
  nav: "训练流程"
  enabled: true

- source: "项目重构/训练流程/train manager在干什么.md"
  dest: "training/train manager在干什么.md"
  title: "Train Manager"
  nav: "训练流程"
  enabled: true
```

这两篇会一起出现在左侧的“训练流程”下面。

## 6. 让几篇内容放在网站同一个文件夹下面

看 `dest` 的前半部分。

例如：

```yaml
dest: "training/PPO_episoid_1训练流程说明书.md"
dest: "training/train manager在干什么.md"
```

它们都会被发布到：

```text
docs_site_staging\docs\notes\training
```

注意：`nav` 控制左侧目录分组，`dest` 控制网站文件夹位置。二者可以一样，也可以不一样。

## 7. 发布整个文件夹

如果 `source` 写的是文件夹，脚本会发布这个文件夹下面的所有文件。

例如：

```yaml
- source: "项目重构/关于robotRUN/RobotRun1_抓取层"
  dest: "project/关于robotRUN/RobotRun1_抓取层"
  title: "RobotRun1 抓取层"
  nav: "RobotRun 分层"
  enabled: true
```

如果文件夹里有：

```text
01_总流程与动作逻辑.md
02_函数说明.md
```

左侧目录会显示为：

```text
RobotRun1 抓取层 / 01_总流程与动作逻辑
RobotRun1 抓取层 / 02_函数说明
```

## 8. 改左侧目录的显示名字

改 `title`。

例如：

```yaml
title: "状态输入"
```

左侧就显示“状态输入”。

## 9. 改左侧目录的分组名字

改 `nav`。

例如：

```yaml
nav: "项目代码说明"
```

所有 `nav: "项目代码说明"` 的页面都会放到这个分组下面。

## 10. 调整显示顺序

左侧目录顺序就是 `publish.yml` 里的顺序。

想让某篇靠前，就把它那一整段移动到前面。

## 11. 本地预览

先同步：

```powershell
cd E:\project_MultiAgent_h_change
python docs_site_staging\scripts\sync_obsidian.py
```

再构建检查：

```powershell
D:\anaconda\envs\rl_env\python.exe -m mkdocs build --strict -f docs_site_staging\mkdocs.yml
```

如果想开本地网页服务：

```powershell
conda activate rl_env
python -m mkdocs serve -f docs_site_staging\mkdocs.yml
```

浏览器打开：

```text
http://127.0.0.1:8000/
```

## 12. 发布到 GitHub Pages

确认本地构建没报错后：

```powershell
git add README.md .github docs_site_staging
git commit -m "更新网页内容"
git push
```

GitHub 会自动运行 Actions 构建网站。

线上地址：

```text
https://shianzhou.github.io/webots-hppo-stair-climbing/
```

如果打不开，去 GitHub 仓库检查：

```text
Settings -> Pages -> Source -> GitHub Actions
```

再去 Actions 页面看 `Deploy MkDocs site` 是否成功。

## 13. 常见问题

### 1. GitHub 上点 `docs_site_staging/site/index.html` 是 404

这是正常的。`site/` 是本地构建产物，不提交到 GitHub。

应该打开：

```text
https://shianzhou.github.io/webots-hppo-stair-climbing/
```

### 2. 改了 Obsidian 笔记，但是网站没变

需要重新运行：

```powershell
python docs_site_staging\scripts\sync_obsidian.py
```

然后重新构建或 push。

### 3. 左侧目录没有变化

左侧目录现在由 `publish.yml` 自动生成。改完 `publish.yml` 后必须重新运行同步脚本。

### 4. 某篇笔记没有出现在网站

检查三件事：

- `source` 路径是否写对。
- `enabled` 是否为 `true`。
- 运行同步脚本时有没有报错。

### 5. 我想保留私人笔记不发布

把它的 `enabled` 设为 `false`，或者干脆不要写进 `publish.yml`。

## 14. 推荐工作流

1. 在 Obsidian 里写笔记。
2. 想发布时，把笔记加入 `publish.yml`。
3. 设置好 `title`、`nav`、`dest`。
4. 运行同步脚本。
5. 本地构建检查。
6. commit + push。
7. GitHub Pages 自动发布。
